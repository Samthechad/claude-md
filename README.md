## What is this?

`CLAUDE.md` is the single most important file in any Claude Code project. It gives Claude persistent context ‚ÄĒ your stack, your conventions, your rules ‚ÄĒ loaded at the start of every session.

**The problem:** Writing it by hand is tedious, most devs skip it, and a bad `CLAUDE.md` is worse than none. Anthropic's own docs warn:

> *"Bloated CLAUDE.md files cause Claude to ignore your actual instructions."*

**The solution:** `claude-md` scans your project, detects your stack, and generates a tight, accurate `CLAUDE.md` in under 10 seconds using Claude AI.

---

## Quickstart

No install needed:

```bash
npx claude-md init
```

Point it at a specific project:

```bash
npx claude-md init ./my-project
```

That's it. Your `CLAUDE.md` is ready.

| **AI** | Anthropic Claude API |

No config required. It just works.

## Your API key is safe

On first run you enter your Anthropic API key once. It's saved to `~/.claude-md/config.json` on **your machine only**. It goes directly from your terminal to Anthropic's API. Nobody else touches it ‚ÄĒ not even us.

Get your key at [console.anthropic.com](https://console.anthropic.com).

## Why the generated CLAUDE.md is actually good

Most CLAUDE.md generators dump everything. This one uses Claude to write Claude instructions ‚ÄĒ which means it knows exactly what Claude Code responds to, what format works, and what to leave out.

The output is:
- **Under 200 lines** ‚ÄĒ short enough that Claude actually reads all of it
- **Stack-specific** ‚ÄĒ no generic filler, only what applies to your project
- **Production-ready** ‚ÄĒ covers commands, conventions, architecture, and rules
